# 迁徙旅行与游戏开发记录系统 Spec

## Why
用户即将开展为期一年、驾驶别克E5从上海出发的全国“追寻舒适气候”的旅行，偏好人文与露营，且每周移动一次（400公里）。同时，用户每天需投入4小时进行独立游戏开发。为了记录这段独特的经历，需要一个适配手机端、简单易用、且具有基础密码保护的个人网页应用，用于打卡地理位置与开发进度，无需复杂的后端数据库。

## What Changes
- 创建一个全新的前端 Web 应用（React + TailwindCSS + Vite）。
- 实现无后端的单密码访问控制（基于 LocalStorage）。
- 实现主面板视图，展示旅行信息（季节迁徙路线提示）与游戏开发目标。
- 实现“地点打卡”功能（记录新地点、营地/城市住宿、人文见闻）。
- 实现“游戏打卡”功能（记录每日开发4小时的完成情况）。
- 所有数据使用 LocalStorage 本地持久化，不依赖外部数据库。
- 采用移动端优先的设计，UI 适配手机访问。

## Impact
- Affected specs: N/A (新项目)
- Affected code: 全新项目，将在 `src/` 下创建所有组件、页面和工具类。

## ADDED Requirements
### Requirement: 认证与安全
系统必须提供一个简单的单密码登录界面，密码硬编码在前端即可，通过验证后才能访问主应用功能。

#### Scenario: 成功登录
- **WHEN** 用户输入正确的密码并提交
- **THEN** 系统记录登录状态至本地缓存，并跳转/渲染至主仪表盘页面

### Requirement: 进度打卡功能
系统必须提供地理位置和游戏开发两种打卡类型，并支持本地查看打卡历史。

#### Scenario: 游戏开发打卡
- **WHEN** 用户在主页点击“完成今日开发”按钮
- **THEN** 系统记录当天的游戏开发完成状态（默认4小时目标），并在历史记录中新增一条数据

#### Scenario: 地点迁移打卡
- **WHEN** 用户在周日移动后，点击“到达新地点”并选择营地/城市
- **THEN** 系统记录新的地理位置信息，更新当前所在地状态
