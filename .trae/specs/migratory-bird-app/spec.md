# 候鸟 (Migratory Bird) App Spec

## Why
用户即将开启为期一年的全国床车旅居（从上海出发，驾驶续航400公里的别克E5纯电车），每周进行移动并在其余时间进行独立游戏开发。市面上的工具要么过于复杂，要么偏向社交，缺乏针对“长期床车旅居 + 数字游民 + 独立开发者”的纯本地化记录应用。因此，需要打造一款名为“候鸟”的专属 App，用于规划随季节变迁的迁徙路线、打卡生活与工作进度，并收集旅途灵感。

## What Changes
- 基于 Android 原生（Jetpack Compose + Material3）从零搭建 MVP 版本。
- 实现纯本地离线的 SQLite 数据库存储（Room），满足“离线优先”的需求。
- 开发核心打卡模块：城市打卡、营地打卡、游戏开发打卡（每天4小时）。
- 开发年度迁徙计划模块：支持按周规划路线及停靠点。
- 开发灵感收集模块。
- 接入地图 SDK 展示足迹。
- 支持将记录和统计数据导出为 Markdown 与 CSV 格式。
- 支持本地图片管理。
- **BREAKING**: 不包含任何社交、聊天、商城和复杂的云同步功能。

## Impact
- Affected specs: 无（全新的核心功能需求）。
- Affected code: 将在 `app/src/main/java/` 目录下建立全新的 MVVM 和 Feature-Based 架构。

## ADDED Requirements
### Requirement: 核心打卡记录系统
系统必须提供城市打卡、营地打卡和游戏开发打卡功能，支持附加图片和备注。
#### Scenario: 游戏开发日常打卡
- **WHEN** 用户完成当天的游戏开发任务
- **THEN** 系统记录打卡时间、时长（默认4小时）、进度备注，并更新项目统计数据。

### Requirement: 年度迁徙计划管理
系统需提供随季节迁徙的年度计划管理，以周为单位（400公里/周）。
#### Scenario: 记录下一站迁徙计划
- **WHEN** 用户规划下一周（周日）的移动路线和目的地（偏好人文/营地/城市住宿）
- **THEN** 系统在计划列表中新增行程，并可在地图上标记预期路线。

### Requirement: 数据导出与本地化管理
系统需完全支持离线使用，图片均保存在本地，支持多格式导出。
#### Scenario: 导出打卡记录
- **WHEN** 用户选择导出特定时间段的记录
- **THEN** 系统生成 Markdown (包含本地图片引用) 或 CSV 文件，并保存至本地存储。
